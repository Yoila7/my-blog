使用技巧

开始对话

  • 1.描述核心功能，不管细节
  • 2.明确技术栈约束
  • 3.拆分核心逻辑，将最难的部分做完再合并进去

迭代过程

  • 1.每次少量调整，方便肉眼审核，避免改变已有功能
  • 2.用具体例子说明现有功能的不足
  • 3.善用视觉反馈
  • 4.开分支做功能，vibe出错了切回主分支即可

context维护

  • 1.AI对久远的context迟钝时，可以让它：“总结一下目前项目的核心架构、已完成的功能和未完成的待办事项”
  • 2.开启新对话，清除废弃思路
  • 3.管理文件可见性，只让AI看与当下任务有关的文件
  • 4.如果AI执着于错误思路，在代码中写注释强制改变逻辑

Skill

提示词+本地工具+工具说明

agent如何调用Skil（渐进式披露）

Skill.md：Skill的名字，基本描述

REFERENCE.md：Skill的使用规则

MCP

AI应用连接外部系统（可以是本地工具，也可以通过API接口连接远程服务器）

提示词工程

  • 1.CoT（分步解决问题）
  • 2.角色扮演
  • 3.格式化输出

上下文管理与代码质量

  • 1.变更代码前先让AI编写测试程序
  • 2.开始全新任务时清除上下文（/clear）
  • 3.对话过长时压缩上下文（/compact）
  • 4.subagent

在大型项目中，为每个子任务指定子代理，各自拥有独立的上下文窗口，在主对话中仅返回任务概要，从而保证对话清爽

Agent Teams

多个Agent一同完成一个任务（任务难以被完全分割为子任务）

Hooks

特定事件触发特定操作（如提交代码前清楚敏感信息）

RAG（检索增强生成）

检索特定领域信息，生成有针对性的回答

规范（Spec）驱动开发

写代码前先让AI确认需求，生成一个贯彻整个项目的Spec

我参考的B站教程

只讲了一点最基本的用法和建立本地仓库以及如何将本地推送至Github
比较全面但是一开始就看可能听不懂，建议学过一些后再看

基础


git init
初始化一个新的 Git 仓库

git log
查看提交历史记录

git status
查看工作区和暂存区的状态

提交


git add #
添加#文件到暂存区

git add .
添加所有修改文件到暂存区

git commit -m "提交信息"
将已放入暂存区的文件提交，并附上说明

git commit -a -m "提交信息"
将已提交的文件的修改和删除操作一步加入暂存区并提交（跳过了add命令，并且无法对新创建的文件生效）

分支相关


git branch #
新建分支

git branch --list
列出所有分支，标绿的是当前分支

git checkout #
切换到#分支

git checkout -b #
新建分支并直接切换到这个分支上

git merge #
将#分支合并到当前分支

解决冲突具体步骤{

git diff
找到冲突的具体内容

编辑存在冲突的文件

git add #
将存在冲突的文件加入暂存区（加入暂存区，文件会被视为冲突已解决）

git commit/git merge --continue
继续合并过程（相当于提交已解决冲突的文件）

}

git merge --abort
发生冲突后可以选择放弃合并，完全回到合并前的状态

git mergetool
打开VScode解决冲突

储存

git stash 保存当前修改，但是不提交（当前还没修改完想去别的分支进行其他修改，但是不保存就无法切换分支）

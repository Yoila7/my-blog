简介

WSL 的全称是 Windows Subsystem for Linux（适用于 Windows 的 Linux 子系统）。

它能让你在不安装虚拟机、不制作双系统的情况下，直接在 Windows 里运行一个原汁原味、速度极快的 Linux 环境。

为什么要用Linux系统

我的理解：作为编程环境而言，Linux内核轻量化，稳定性强。

为什么选择WSL

感觉双系统和虚拟机都好麻烦啊

WSL有占用硬盘空间少，启动速度快，Windows和Linux系统同时运行无缝切换的优势。

WSL的使用体验

它没有系统本身的GUI，只能用CLI的方式与它交互（打开Windows Terminal选择名为Ubuntu的Shell）。因此不会有想象中的全新的系统界面，你实际上还在Windows系统之中，在你的Windows系统中运行着Linux的内核，同时你仍然可以做所有你本来在Windows系统中可以做的任何事。安装完插件后你甚至可以打开VScode自动运行WSL然后在VScode中的编程就已经是在Linux环境下的了（后面会讲）。

就像重装完系统，此时你的Linux是一个白板，你在Windows系统中安装的所有东西都不会存在，所以要在WSL中进行一系列的安装和配置。

VScode的WSL扩展：打开VScode，在扩展中查找WSL并安装，然后利用这个扩展连接到你的WSL。
官方网页
（这个扩展的意义：在Windows系统下打开VScode，像原本在Windows系统下一样编程，但是Vscode会将你的代码传递给WSL保存，编译，运行。连接到WSL后，在VScode中新建/移动文件、文件夹等操作就直接作用于你的WSL系统，这样你就不需要记忆这些命令行操作了，直接在VScode中就可以完成。总之是VScode在调用WSL，VScode本身还是在Windows系统下，因此WSL中没有也不必安装VScode。）
